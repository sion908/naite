# NaITE ホームページ運用ガイド

このドキュメントでは、ローカルでの動作確認方法と、イベント情報の追加・更新手順について説明します。

## 1. ローカルでの動作確認 (Execution Check)

開発中のサイトを自分のパソコン（ローカル環境）で確認する手順です。

### 必要なもの
- Node.js (推奨: v18以上)
- ターミナル (Macのターミナル、VSCodeのターミナルなど)

### 手順
1. **依存関係のインストール** (初回のみ)
   まだ行っていない場合は、プロジェクトのルートディレクトリで以下のコマンドを実行します。
   ```bash
   npm install
   ```

2. **開発サーバーの起動**
   以下のコマンドを実行します。
   ```bash
   npm run dev
   ```

3. **ブラウザで確認**
   ターミナルに `Local: http://localhost:4321/` のような表示が出ます。
   ブラウザで [http://localhost:4321](http://localhost:4321) を開くと、現在のサイトの状態を確認できます。
   ファイルを保存すると、ブラウザは自動的にリロードされます。

4. **ビルド確認** (公開前の最終確認)
   本番環境と同じ静的ファイルを生成して確認したい場合は、以下を実行します。
   ```bash
   npm run build
   npm run preview
   ```
   `preview` コマンドで起動したURLで、ビルド後の挙動を確認できます。

## 2. イベントの追加手順 (Adding New Events)

新しいイベントが決まった際に、イベントページに追加する手順です。
Markdownファイルを作成するだけで、自動的に一覧ページと詳細ページが生成されます。

### ファイルの作成場所
`src/content/events/` ディレクトリの中に新しいファイルを作成します。

ファイルの命名規則: `YYYY-MM-DD-イベント名.md`
例: `2026-05-20-study-meeting.md`

### ファイルの書き方

作成した `.md` ファイルに、以下の形式で内容を記述してください。

```markdown
---
title: "イベントのタイトル"
description: "イベントの簡単な説明（一覧ページに表示されます）"
date: 2026-05-20
location: "開催場所（例：出島メッセ長崎 / オンライン）"
links:
  - text: "Connpassページ"
    url: "https://connpass.com/..."
  - text: "資料URL"
    url: "https://..."
---

## イベント概要
ここにイベントの詳細な説明を書きます。Markdown記法が使えます。

- 箇条書き
- **太字**

## タイムテーブル
...
```

### 必須項目 (Frontmatter)
ファイルの先頭にある `---` で囲まれた部分（フロントマター）は必須です。

| 項目名 | 説明 | 例 |
| :--- | :--- | :--- |
| `title` | イベント名 | `"第10回 NaITE勉強会"` |
| `description` | 短い説明文 | `"Reactについて学ぶ会です"` |
| `date` | 開催日 | `2026-05-20` (YYYY-MM-DD形式) |

### オプション項目
| 項目名 | 説明 |
| :--- | :--- |
| `location` | 開催場所 |
| `links` | 関連リンクのリスト (`text` と `url` のペア) |

## 3. サイトの更新・公開手順

### 手順
1. **変更をコミットする**
   ```bash
   git add .
   git commit -m "イベント情報を追加: 第10回勉強会"
   ```

2. **GitHubにプッシュする**
   ```bash
   git push origin main
   ```

3. **自動デプロイ**
   プッシュすると、GitHub Actionsが自動的に動き出し、数分後には公開サイト (GitHub Pages) が更新されます。
   GitHubリポジトリの "Actions" タブで進捗を確認できます。
